> **为何两个口径差 136 倍？**
> state.db 只记每次 API 响应的**新增** tokens；DeepSeek 按每次请求的**完整上下文窗口**计费（含之前所有轮次累积的历史）。14.08B / 80,968 ≈ 174K tokens/请求，即平均每次都携带了约 174K 的上下文。

## 产品架构

### 表现层 · Presentation
- Flutter UI Widget — lib/pages/ 底栏 4Tab + 解题页/讲义/组卷等子页面
- HTML 原型 — spec/UI_html/ 30+ 文件，data-db 标记体系 → Flutter Widget 转换工作流
- 路由规范 — 集中路由表 router.dart，参数化路由 /solve?qid=&from=
- 常量/文案体系 — _Strings / _Prefs / app_strings.dart / app_prefs.dart
- UI 辅助组件 — 版本更新弹窗、下载进度条、解题冷却倒计时、评分/评价弹窗

### 领域层 · Repository / Domain
- 13 个 Repository：AuthRepository、AssignmentRepository、LectureRepository、QuestionRepository、ProgressRepository、RatingRepository、SyncRepository、PreferenceRepository 等
- 5 个算法引擎（私有尾部类）：_PointsCalculator（积分）、_AchievementEngine（成就）、_StatisticsAggregator（统计）、_RecommendationEngine（推荐）、_ExamFilterEngine + _ExamGenerator（组卷）

### 数据访问层 · Data Access
- Drift DAO — 表级 CRUD 封装，三库读写（assets_db / lectures_db / user_db）
- ApiClient — HTTP 封装，JWT 自动刷新 · 401 重定向 · 统一错误处理
- SharedPrefs Wrapper — SharedPreferences 封装
- ConnectivityMonitor — connectivity_plus 网络状态监听

### 本地持久化层 · Local Storage
- SQLite (Drift) — 三库独立（app.db / lectures.db / assets.db）
- 图片存储 — 题库 WebP（Flutter assets）+ 用户头像（cached_network_image）
- SharedPreferences — token、版本号、冷却、引导状态、评价冷却
- sync_queue 表 — 待推送操作队列（sync_type / payload）

### 通信基础设施层 · Communication
- 同步引擎 — SyncManager 协调 · SyncPusher 队列推送 · 失败重试 · LWW 冲突解决
- 更新机制 — VersionChecker · .db.gz 下载+校验 · schema 版本 3 版强制 · 文件替换

### 服务端层 · Server
- Django REST API（/api/v1/）— 5 个功能 App：accounts / qbank / courses / interactions / system
- Django Admin（/admin/）— 教师/管理员操作入口，CRUD + 批量操作
- 构建脚本 — Django data → assets.db / lectures.db 构建工具

### 基础设施层 · Infrastructure
- nginx — 反向代理 · .db.gz 直出 · 着陆页直出
- Landing 着陆页 — 下载 · 微信 · 隐私/用户协议
- Cloudflare Tunnel — TLS 终结 · 免证书维护
- ECS（2C2G · 阿里云）

### 测试（横切各层）
单测 → 集成 → UI → 离线 → E2E，覆盖全部7层

> 跨层标记体系：**data-db** 标记（HTML ↔ Repository 映射），贯穿设计→编码全流程


### 数据流

```
登录 → 全量拉取 user.db.gz（gzip 压缩 SQLite）
运行 → 本地 SQLite（优先）→ 写入 sync_queue → 异步推送
更新 → 版本号检查 → .db 整库替换（讲义/题库/assets 分别管理）
```

## 关键决策回顾

| 决策 | 路径 A（被否） | 路径 B（执行） | 代价 |
|------|--------------|--------------|------|
| **PDF 方案** | Flutter 端渲染 | 服务端 HTML → 浏览器打印 | 3 次迭代，数天开销 |
