### 设计期要事

**PDF 方案三次迭代（7/9）：**
1. Flutter 客户端渲染（`_ExamPdfService`）— 被否
2. 服务端 HTML → 浏览器打印 — 定稿
3. 客户端只留入口 + 引导弹窗 — 最终形态

**教师端决策（7/10）：** 在"纯 Web"和"Flutter 独立 App"之间选择了**混合方案**：
- 教师使用 Flutter 学生端 App 的部分功能 + 轻量 Web 仪表盘
- 这个决策后来导致了大量代码复用问题（催生 shared 包）

**数据架构定稿：**
- 三库分离：`app.db`（用户数据）/ `lectures.db`（讲义）/ `assets.db`（配图）
- 同步引擎：登录全量拉取 + 增量 sync_queue 异步推送
- 方案 B：讲义客户端镜像存储 raw markdown，渲染时解析分隔符
